# Установка

[Вернуться к README](../README.md)  
[Перейти к инструкции по использованию](./USAGE.md)

Этот документ описывает установку текущей remote-версии `ChatGPT Web Bridge`: сервера, браузерного расширения и базовой конфигурации для дальнейшего подключения MCP-клиентов.

## Что будет установлено

После завершения установки у вас будет:

- сервер `chatgpt-web-bridge`, принимающий MCP-клиентов на `/mcp`;
- браузерное расширение, работающее как browser-agent и подключающееся к `/agent/ws`;
- общий набор токенов, связывающий ваши MCP-клиенты и агент в один пользовательский namespace.

## Требования

- Windows с PowerShell;
- Node.js `24+`;
- Firefox `121+` или совместимый Gecko-браузер;
- действующая авторизация в `https://chatgpt.com/`;
- MCP-клиент, умеющий работать со `Streamable HTTP`.

## 1. Подготовить проект

```powershell
cd D:\development\projects\chatgpt-web-bridge
npm install
npm run build
```

Проверка сборки:

```powershell
npm run typecheck
npm test
```

## 2. Выбрать токены

Перед запуском определите два значения:

- `server access token`: секрет конкретного серверного инстанса;
- `user token`: общий пользовательский токен, одинаковый для extension и MCP-клиентов этого пользователя.

Рекомендации:

- используйте длинные случайные значения;
- не публикуйте их в логах и примерах;
- для разных пользователей используйте разные `user token`;
- если вы хотите полностью изолировать другой набор клиентов и вкладок, используйте другой `user token`.

## 3. Запустить сервер

Минимальный запуск на `localhost`:

```powershell
npm run start:server -- --host=127.0.0.1 --port=8787 --server-access-token=replace-me
```

Эквивалент напрямую:

```powershell
node dist/index.js --transport=server --host=127.0.0.1 --port=8787 --server-access-token=replace-me
```

Сервер стартует в HTTP-режиме и выводит адрес вида:

```text
[server] Listening on http://127.0.0.1:8787
```

Быстрая проверка доступности:

```powershell
curl http://127.0.0.1:8787/health
```

Ожидаемый ответ содержит `ok`, число подключенных агентов и количество активных сессий.

## 4. Установка расширения

### Вариант A. Временная загрузка в Firefox

Это основной путь для разработки и локальной проверки.

1. Откройте `about:debugging#/runtime/this-firefox`
2. Нажмите `Load Temporary Add-on...`
3. Выберите файл `...\chatgpt-web-bridge\extension\manifest.json`
4. Откройте `https://chatgpt.com/`
5. Убедитесь, что вы уже вошли в аккаунт

### Вариант B. Сборка одним файлом `.xpi`

Если нужен один файл расширения:

```powershell
npm run build:addon:unsigned
```

После этого появится unsigned `.xpi`:

```text
D:\development\projects\chatgpt-web-bridge\artifacts\unsigned-addon\chatgpt-web-bridge-0.1.0-unsigned.xpi
```

Важно:

- это unsigned addon;
- для Firefox stable он подходит только для временной загрузки/отладки;
- для dev-установки в отдельный профиль в репозитории есть Waterfox workflow.

### Вариант C. Waterfox dev-профиль

Если нужен готовый dev-профиль c установленным unsigned addon:

```powershell
npm run install:waterfox-dev
```

Дальше можно запустить браузер так:

```powershell
npm run run:waterfox-dev
```

## 5. Настроить browser-agent

После загрузки расширения откройте его страницу настроек и заполните три поля:

- `Server URL`
- `Server Access Token`
- `User Token`

Для локальной схемы значения обычно такие:

- `Server URL`: `http://127.0.0.1:8787`
- `Server Access Token`: тот же токен, что вы передали серверу
- `User Token`: ваш пользовательский токен

После сохранения в статусе расширения должно появиться состояние `ready`.

Если настроек еще нет, нажатие на иконку расширения открывает страницу options автоматически.

## 6. Выбрать схему подключения

Одна и та же установка поддерживает несколько топологий.

### Полностью локально

- сервер: `http://127.0.0.1:8787`
- extension: `http://127.0.0.1:8787`
- MCP-клиенты: `http://127.0.0.1:8787/mcp`

### Локальный сервер + Cloudflare Tunnel

- сервер слушает локально, например `127.0.0.1:8787`
- extension может подключаться к `http://127.0.0.1:8787`
- MCP-клиенты могут подключаться к tunnel/public URL того же сервера

Это штатная конфигурация: агент и клиенты все равно встретятся внутри одного сервера по одинаковым токенам.

### Все через внешний URL

- extension использует tunnel/public URL
- MCP-клиенты используют тот же внешний URL

Этот вариант тоже поддерживается без отдельной серверной ветки.
